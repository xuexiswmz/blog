# CSS 隐藏元素：不同方案的布局与交互差异

你有一个弹窗，关闭时要隐藏：

```css
.modal {
  display: none; /* 还是 */
  visibility: hidden; /* 还是 */
  opacity: 0; /* ？ */
}
```

它们都能隐藏，但行为完全不同：

- `display: none` — 元素彻底消失，不占空间
- `visibility: hidden` — 元素隐藏，但位置还在
- `opacity: 0` — 元素透明，事件还能触发

选错了，布局会乱，或者点击穿透。

---

## 1. display: none

**效果：** 元素从文档流中移除，像从来没存在过。

```css
.box {
  display: none;
}
```

**特点：**

| 特性 | 值 |
|---|---|
| 占据空间 | ❌ 不占 |
| 触发重排 | ✅ 会 |
| 触发重绘 | ✅ 会 |
| 事件触发 | ❌ 不能 |
| 过渡动画 | ❌ 不支持 |
| 子元素显示 | ❌ 不能（子元素也隐藏） |

**更适合：**

- 彻底移除元素（如切换 tab）
- 不需要动画的显示/隐藏

**注意：** 无法直接对 `display` 做过渡，因为元素会立即从文档流中移除。

```css
.box {
  transition: opacity 0.3s;
  display: none; /* 过渡无效！ */
}
```

---

## 2. visibility: hidden

**效果：** 元素隐藏，但空间保留。

```css
.box {
  visibility: hidden;
}
```

**特点：**

| 特性 | 值 |
|---|---|
| 占据空间 | ✅ 占据 |
| 触发重排 | ❌ 不会 |
| 触发重绘 | ✅ 会 |
| 事件触发 | ❌ 不能 |
| 过渡动画 | ✅ 支持 |
| 子元素显示 | ✅ 能（子元素设置 `visible`） |

**子元素可以显示：**

```css
.parent {
  visibility: hidden;
}

.child {
  visibility: visible; /* 子元素可见 */
}
```

**更适合：**

- 需要保留布局位置
- 需要过渡动画
- 父隐藏但子显示

---

## 3. opacity: 0

**效果：** 元素透明，但完全存在。

```css
.box {
  opacity: 0;
}
```

**特点：**

| 特性 | 值 |
|---|---|
| 占据空间 | ✅ 占据 |
| 触发重排 | ❌ 不会 |
| 触发重绘 | ⚠️ 可能 |
| 事件触发 | ✅ 能 |
| 过渡动画 | ✅ 支持 |
| 子元素显示 | ❌ 不能（子元素也透明） |

**注意：** 元素透明后仍可能接收鼠标和触摸事件；如果不希望它参与交互，需要配合 `pointer-events` 等处理。

```css
.button {
  opacity: 0;
}

/* 点击透明区域，按钮还是会被点击！ */
```

**解决：** 加 `pointer-events: none`

```css
.button {
  opacity: 0;
  pointer-events: none; /* 禁用事件 */
}
```

**更适合：**

- 淡入淡出动画
- 临时隐藏但保留交互能力

---

## 4. position: absolute + 移出视口

**效果：** 把元素移到屏幕外。

```css
.box {
  position: absolute;
  left: -9999px;
  top: -9999px;
}
```

**特点：**

| 特性 | 值 |
|---|---|
| 占据空间 | ❌ 不占（脱离文档流） |
| 触发重排 | ✅ 会 |
| 事件触发 | ❌ 不能（在视口外） |
| 过渡动画 | ⚠️ 可以（移动动画） |

**更适合：**

- 隐藏但对屏幕阅读器可见（无障碍）
- 临时移出视口

---

## 5. height: 0 + overflow: hidden

**效果：** 把元素高度压成 0。

```css
.box {
  height: 0;
  overflow: hidden;
  padding: 0;
  margin: 0;
  border: none;
}
```

**特点：**

| 特性 | 值 |
|---|---|
| 占据空间 | ❌ 不占 |
| 触发重排 | ✅ 会 |
| 事件触发 | ❌ 不能 |
| 过渡动画 | ✅ 支持（高度动画） |

**更适合：**

- 折叠/展开动画
- 手风琴效果

```css
.box {
  height: 0;
  overflow: hidden;
  transition: height 0.3s;
}

.box.expanded {
  height: 200px;
}
```

---

## 6. clip-path: polygon(0 0, 0 0, 0 0, 0 0)

**效果：** 裁剪区域为空，视觉上隐藏。

```css
.box {
  clip-path: polygon(0 0, 0 0, 0 0, 0 0);
}
```

**特点：**

| 特性 | 值 |
|---|---|
| 占据空间 | ✅ 占据 |
| 触发重排 | ❌ 不会 |
| 事件触发 | ❌ 不能 |
| 过渡动画 | ✅ 支持 |

**更适合：**

- 创意隐藏动画
- 元素"消失"动画

## 7. 五种方式对比

| 方式 | 占空间 | 重排 | 重绘 | 事件 | 过渡 |
|------|--------|------|------|------|------|
| `display: none` | ❌ | ✅ | ✅ | ❌ | ❌ |
| `visibility: hidden` | ✅ | ❌ | ✅ | ❌ | ✅ |
| `opacity: 0` | ✅ | ❌ | ⚠️ | ✅ | ✅ |
| `position` 移出 | ❌ | ✅ | - | ❌ | ⚠️ |
| `height: 0` | ❌ | ✅ | - | ❌ | ✅ |
| `clip-path` | ✅ | ❌ | - | ❌ | ✅ |

## 8. 放到具体需求里看

| 需求 | 常见选择 |
|------|---------|
| 彻底隐藏，不占空间 | `display: none` |
| 隐藏但保留位置 | `visibility: hidden` |
| 淡入淡出动画 | `opacity: 0` + `pointer-events: none` |
| 折叠展开动画 | `height: 0` + `overflow: hidden` |
| 无障碍隐藏 | `position: absolute` + 移出视口 |

**最常见的组合：**

- **`display: none`**：彻底隐藏
- **`visibility: hidden`**：保留位置
- **`opacity: 0`**：淡入淡出

---

## 9. 回到开头的弹窗

**几个差异值得放在一起看：**

1. **`display: none`** — 元素消失，不占空间，重排
2. **`visibility: hidden`** — 元素隐藏，占空间，重绘
3. **`opacity: 0`** — 元素透明，占空间，事件可触发

**实际选择时：**

需要淡入淡出时，`opacity` 或 `visibility` 更容易参与过渡；元素关闭后不再占空间，通常会用 `display: none`；布局位置还要保留，则更接近 `visibility: hidden` 的行为。
