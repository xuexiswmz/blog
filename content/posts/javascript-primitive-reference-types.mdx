# JavaScript 基本类型与引用类型：为什么复制后的值表现不同

看这段代码：

```javascript
let a = 1;
let b = a;
a = 2;
console.log(b); // 1 ← 没变
```

再看看这段：

```javascript
let arr1 = [1, 2, 3];
let arr2 = arr1;
arr1.push(4);
console.log(arr2); // [1, 2, 3, 4] ← 变了！
```

为什么 `b` 不受影响，`arr2` 却跟着变了？

因为 `a` 是**基本类型**，`arr1` 是**引用类型**。它们的存储方式完全不同。

---

## 1. 两种类型的本质区别

### 基本类型：赋值时复制值

```javascript
let a = 1;
```

从语言行为上看，变量 `a` 保存的是值 `1`；把它赋给另一个变量时，会复制这个值。

```
┌─────┐
│  a  │ → 1
└─────┘
```

### 对象类型：赋值时共享引用

```javascript
let arr = [1, 2, 3];
```

从语言行为上看，变量 `arr` 持有对数组对象的引用。把 `arr` 赋给另一个变量时，两个变量会指向同一个数组。

```
┌─────┐      ┌───────────────┐
│ arr │ ───→ │ [1, 2, 3]     │ 堆内存
└─────┘      └───────────────┘
```

## 2. JavaScript 的数据类型

### 基本类型（7 种）

| 类型 | 说明 | 示例 |
|------|------|------|
| `Number` | 数字（整数和浮点数） | `1`, `3.14`, `NaN`, `Infinity` |
| `String` | 字符串 | `'hello'`, `"world"` |
| `Boolean` | 布尔值 | `true`, `false` |
| `undefined` | 未定义 | `undefined` |
| `null` | 空值 | `null` |
| `Symbol` | 唯一标识符（ES6） | `Symbol('id')` |
| `BigInt` | 大整数（ES2020） | `9007199254740991n` |

### 引用类型

| 类型 | 说明 | 示例 |
|------|------|------|
| `Object` | 对象 | `{ name: 'Alice' }` |
| `Array` | 数组 | `[1, 2, 3]` |
| `Function` | 函数 | `function() {}` |
| `Date` | 日期 | `new Date()` |
| `RegExp` | 正则 | `/pattern/` |
| `Map` / `Set` | 集合（ES6） | `new Map()` |

**注意：** 数组、函数本质上都是对象。

## 3. 赋值行为的差异

### 基本类型：值复制

```javascript
let a = 1;
let b = a; // 复制值

// a 和 b 是独立的
a = 2;
console.log(b); // 1 ← b 不受影响
```

```
赋值前：
┌─────┐
│  a  │ → 1
└─────┘

赋值后：
┌─────┐
│  a  │ → 1
└─────┘
┌─────┐
│  b  │ → 1（复制了一份）
└─────┘

修改 a：
┌─────┐
│  a  │ → 2
└─────┘
┌─────┐
│  b  │ → 1（不变）
└─────┘
```

### 引用类型：地址复制

```javascript
let arr1 = [1, 2, 3];
let arr2 = arr1; // 复制地址

// arr1 和 arr2 指向同一个数组
arr1.push(4);
console.log(arr2); // [1, 2, 3, 4] ← 变了！
```

```
赋值前：
┌──────┐      ┌─────────────┐
│ arr1 │ ───→ │ [1, 2, 3]   │
└──────┘      └─────────────┘

赋值后：
┌──────┐
│ arr1 │ ──┐
└──────┘   │    ┌─────────────┐
           └──→ │ [1, 2, 3]   │
┌──────┐   ┌    └─────────────┘
│ arr2 │ ──┘
└──────┘

修改 arr1：
┌──────┐
│ arr1 │ ──┐
└──────┘   │    ┌─────────────────┐
           └──→ │ [1, 2, 3, 4]    │ ← 变了
┌──────┐   ┌    └─────────────────┘
│ arr2 │ ──┘
└──────┘
```

## 4. 函数参数传递

### 基本类型：传值

```javascript
function changeValue(x) {
  x = 100; // 修改的是副本
}

let num = 1;
changeValue(num);
console.log(num); // 1 ← 不变
```

### 引用类型：传引用

```javascript
function changeArray(arr) {
  arr.push(4); // 修改的是原数组
}

let nums = [1, 2, 3];
changeArray(nums);
console.log(nums); // [1, 2, 3, 4] ← 变了
```

**坑：重新赋值不会影响原变量**

```javascript
function reassign(arr) {
  arr = [4, 5, 6]; // 重新赋值，断开引用
}

let nums = [1, 2, 3];
reassign(nums);
console.log(nums); // [1, 2, 3] ← 不变
```

`arr = [4, 5, 6]` 让参数 `arr` 指向新数组，不影响外部的 `nums`。

## 5. 类型检测

### typeof

```javascript
typeof 1;           // 'number'
typeof 'hello';     // 'string'
typeof true;        // 'boolean'
typeof undefined;   // 'undefined'
typeof Symbol();    // 'symbol'
typeof 1n;          // 'bigint'

typeof null;        // 'object' ← 历史遗留 bug！
typeof {};          // 'object'
typeof [];          // 'object' ← 数组也是 object
typeof (() => {});  // 'function'
```

### 判断数组

```javascript
Array.isArray([1, 2, 3]); // true ✅
typeof [1, 2, 3];         // 'object' ❌
[1, 2, 3] instanceof Array; // true
```

### 判断 null

```javascript
value === null; // ✅ 最可靠
typeof value === 'object' && value === null; // 也可以
```

## 6. 如何理解“栈与堆”模型

教学资料常用“基本类型放在栈、对象放在堆”来帮助理解复制行为。这个模型很直观，但 JavaScript 规范并没有规定引擎必须采用某一种内存布局；具体实现还会受到优化、逃逸分析和垃圾回收策略影响。

```javascript
// 基本值：赋值后彼此独立
let a = 1;
let b = 2;

// 对象：变量持有引用
let obj = { x: 1, y: 2 };
let arr = [1, 2, 3];
```

实际写代码时，更可靠的判断方式是关注语言语义：基本值按值复制，对象按引用共享；对象的生命周期由垃圾回收器管理。

## 7. 几个容易混淆的地方

### 情况 1：比较引用

```javascript
const a = [1, 2];
const b = [1, 2];

console.log(a === b); // false ← 引用不同
console.log(a == b);  // false
```

即使内容一样，引用类型比较的是地址。

### 情况 2：对象做函数默认参数

```javascript
function addItem(item, list = []) {
  list.push(item);
  return list;
}

console.log(addItem('a')); // ['a']
console.log(addItem('b')); // ['b'] ← 不是 ['a', 'b']！
```

每次调用都创建新的默认参数，不会共享。

### 情况 3：const 不能阻止修改

```javascript
const arr = [1, 2, 3];
arr.push(4); // ✅ 可以修改内容

arr = [4, 5, 6]; // ❌ 不能重新赋值
```

`const` 保证变量指向的地址不变，不保证内容不变。

## 8. 回到复制行为

| | 基本类型 | 引用类型 |
|---|---------|---------|
| 赋值语义 | 复制值 | 复制对象引用 |
| 比较 | 比较值 | 比较是否为同一引用 |
| 修改 | 不影响其他变量 | 影响共享引用的变量 |
| 例子 | `number`, `string`, `boolean` | `object`, `array`, `function` |

**简单收一下：**
- 基本类型：值复制，互不影响
- 对象类型：引用复制，共享数据
- 要独立复制引用类型，需要深拷贝
